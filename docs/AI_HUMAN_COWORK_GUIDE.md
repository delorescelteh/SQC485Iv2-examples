# AI + Human 協作指南（SQC485Iv2 / Meshtastic）

目的：讓使用者在 DIY、除錯、改韌體、寫文件時，能用「AI + 人」的方式更快更穩。

## 0) 核心原則

- **AI 擅長**：整理、比對、生成草稿、列測試清單、找規律、改文件、做腳本。
- **人擅長**：碰硬體、確認按鍵/接線、觀察 LED/聲音、判斷是否安全、決定要不要公開。
- **先可重現、再優化**：每次只改一個變因。
- **不要把秘密丟給 AI**：WiFi 密碼、Token、私鑰、住址電話等。

## 1) 提問模板（最有用）

把下面資訊貼給 AI（越完整越快）：

- 目標：例如「刷入韌體」「兩台互通」「做 release」
- 硬體：板子版本、是否接天線、供電方式（USB/太陽能/電池）
- 軟體：Meshtastic 版本、env 名稱、OS
- 現象：你看到的 log / 錯誤訊息（完整貼出）
- 你做過什麼：已嘗試步驟

## 2) 硬體相關任務：人要做的 5 個確認

1. 天線是否接好（SX1262 無天線發射有風險）
2. 兩台距離先拉近（1–3 公尺）做互通
3. 供電是否穩（避免 USB hub / 線材問題）
4. 是否同一個 Region/Channel/PSK
5. 是否有按鍵/bootloader 操作需要手動介入

## 3) AI 可以做的任務清單

- 產生「一步一步」刷機/測試 SOP
- 把 log 轉成「可能原因排名」
- 寫腳本（批次燒錄、批次測試）
- 寫 Release notes、SHA256 檢查檔
- 把原理圖文字整理成 pinout 表格（再由人確認）

## 4) 共同工作流程（建議）

1. 人：描述問題 + 提供 log
2. AI：給 3–5 個最高機率原因 + 最短測試路徑
3. 人：照做並回報結果
4. AI：收斂、更新文件/範例
5. 最終：把可重現流程寫進 repo（大家都能用）

## 5) 開源/公開的注意事項

- 不要公開任何 secret（token、WiFi、私鑰）
- Release 放：`.bin`、`.factory.bin`、`SHA256SUMS.txt`
- 文件要說清楚：適用硬體、Region、刷機方式、風險（尤其供電/電池）
